## PrDiOffAmp is a primer-dimers and off-target amplification detection tool.

# Usage
PrDiOffAmp is a command-line tool and can be run with the following command
python3 PrDiOffAmp_analyze.py -i {amplicon_id} -1 {path_to_paired_end_fastq_file_1} -2 {path_to_paired_end_fastq_file_2} -p {path_to_primers.fasta} --plot {path_to_save_plot.png} -o {path_to_save_summary.tsv}

Help can be called with

python3 PrDiOffAmp_analyze.py -h

# Options
  * `-h`, `--help`            show this help message and exit
    Required arguments:
  * `-i SAMPLE_ID`, `--sample-id SAMPLE_ID`
                         <br /> &emsp;&emsp;&emsp; Provide sample identifier (used in the summary table)
  *  `-1 R1`, `--r1 R1`,
                        <br /> &emsp;&emsp;&emsp; Provide path to forward FASTQ (R1). Supports .gz 
  *  `-2 R2`, `--r2 R2`,
                        <br /> &emsp;&emsp;&emsp; Provide path to reverse FASTQ (R2). Supports .gz
  *  `-p PRIMERS`, `--primers PRIMERS`, 
                        <br /> &emsp;&emsp;&emsp; Provide path to FASTA file containing forward and reverse primers"
    Optional arguments:
  *  `-o OUT`, `--out OUT`, 
                        <br /> &emsp;&emsp;&emsp; Provide path to to write the tab‑separated summary (default: summary.tsv)
  *  `--expected-len EXPECTED_LEN`, 
                        <br /> &emsp;&emsp;&emsp; Give expected reads length range (default: 600, for merged reads)
  *  `--tolerance TOLERANCE`, 
                        <br /> &emsp;&emsp;&emsp; Give length tolerance around the expected length (default: 50) for off-target aplification detection
  *  `--max-dimer-len MAX_DIMER_LEN`, 
                        <br /> &emsp;&emsp;&emsp; Give maximum length to consider a read a primer‑dimer (default: 100*2, for merged reads)
  *  `--max-mismatches MAX_MISMATCHES`, 
                        <br /> &emsp;&emsp;&emsp; Give maximum number of mismatches allowed when checking primer ends (default: 2) for primer-dimers detection
  *  `--plot PLOT`, 
                        <br /> &emsp;&emsp;&emsp; Provide path to write a histogram PNG/PDF to this path

# Input expectations and default settings
Standard fastq is expected for reads, including the PHRED quality score line. As it is dealing with paired-end sequencing, it is assumed that number in the pair is indicated according to Illumina format convention (using /1 and /2).
Default settings assume 300 bp reads.

# How to read the output

* summary.tsv. This is a tab-separeted file containing all the main statistics generated by the tool. Per sample (see sample_id) the information on total number of reads (total_reads), number of primer-dimers (primer_dimer_count), the corresponding percentage (primer_dimer_percentage), the number of off-target amplification sequences shorter than the tolerable leght (short_offtarget_count), and longer (long_offtarget_count) and the number of valid sequences (valid_amplicon_count) are stored.
* plot.png. This file shows (per sample) the histogram of read length distribution. In the background, coloured regions mark primer dimers and off-target products length ranges.

# Example workflow

1) Install requirements 
`pip install -r requirements.txt`
Alternatively requirements can be installed with conda
`conda create -n detection_env python=3.10`
`conda activate detection_env`
`conda install -c conda-forge biopython pandas matplotlib numpy tqdm`
2) Run the helper script to generate simple simulated data
`python3 generate_fastq.py`
3) Run PrDiOffAmp
`python3 PrDiOffAmp_analyze.py -i amplicon -1 R1.fastq -2 R2.fastq -p primers.fasta --plot plot1.png -o summary.tsv`
4) Look at the output
